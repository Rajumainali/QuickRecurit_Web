# SentenceTransformer based on sentence-transformers/all-MiniLM-L6-v2

This is a [sentence-transformers](https://www.SBERT.net) model finetuned from [sentence-transformers/all-MiniLM-L6-v2](https://huggingface.co/sentence-transformers/all-MiniLM-L6-v2). It maps sentences & paragraphs to a 384-dimensional dense vector space and can be used for semantic textual similarity, semantic search, paraphrase mining, text classification, clustering, and more.

## Model Details

### Model Description
- **Model Type:** Sentence Transformer
- **Base model:** [sentence-transformers/all-MiniLM-L6-v2](https://huggingface.co/sentence-transformers/all-MiniLM-L6-v2) <!-- at revision c9745ed1d9f207416be6d2e6f8de32d1f16199bf -->
- **Maximum Sequence Length:** 256 tokens
- **Output Dimensionality:** 384 dimensions
- **Similarity Function:** Cosine Similarity

### Full Model Architecture

```
SentenceTransformer(
  (0): Transformer({'max_seq_length': 256, 'do_lower_case': False, 'architecture': 'BertModel'})
  (1): Pooling({'word_embedding_dimension': 384, 'pooling_mode_cls_token': False, 'pooling_mode_mean_tokens': True, 'pooling_mode_max_tokens': False, 'pooling_mode_mean_sqrt_len_tokens': False, 'pooling_mode_weightedmean_tokens': False, 'pooling_mode_lasttoken': False, 'include_prompt': True})
  (2): Normalize()
)
```

## Usage

### Direct Usage (Sentence Transformers)

First install the Sentence Transformers library:

```bash
pip install -U sentence-transformers
```

Then you can load this model and run inference.
```python
from sentence_transformers import SentenceTransformer

# Download from the ðŸ¤— Hub
model = SentenceTransformer("sentence_transformers_model_id")
# Run inference
sentences = [
    'Seasoned Database Administrator with six years of experience in PostgreSQL environments. Expert in performance tuning, disaster recovery, and database security. Strong communication and problem-solving skills for enterprise-level database management.\nSQL, PostgreSQL, Performance Tuning, Backup/Recovery, Database Security, Communication, Problem-Solving\nOptimized PostgreSQL databases, achieving a 70% reduction in query time. Developed robust backup systems, ensuring 100% data availability. Implemented advanced security protocols, eliminating 90% of vulnerabilities.\nBachelor of Science in Computer Science, LMN University, 2018',
    " Database Administrator (DBA) with a Bachelor's in Computer Science/IT and 2+ years of experience in database management, who is proficient in SQL and database systems (MySQL/PostgreSQL/SQL Server), skilled in backup/recovery, knowledgeable in performance tuning, understands database security, and possesses strong problem-solving, attention to detail, and communication skills.",
    "Software Engineer (Backend) with a Bachelor's in Computer Science or related field and 2+ years of experience in backend development, who is proficient in Python/Java/Node.js, experienced with RESTful APIs, knowledgeable in database management (SQL/NoSQL), familiar with cloud platforms (AWS/Azure/GCP), understands version control (Git), and possesses strong problem-solving, debugging, and communication skills.",
]
embeddings = model.encode(sentences)
print(embeddings.shape)
# [3, 384]

# Get the similarity scores for the embeddings
similarities = model.similarity(embeddings, embeddings)
print(similarities)
# tensor([[1.0000, 0.9171, 0.8688],
#         [0.9171, 1.0000, 0.9470],
#         [0.8688, 0.9470, 1.0000]])
```
